# Codex × GPT Image 2 提示詞分享版

這份文件整理一組可公開分享的提示詞範本，適合放在 GitHub，作為 Codex 與 GPT Image 2 協作的實作參考。

使用方式：直接複製需要的提示詞，替換中括號內容即可。範例以公開技術分享、專案案例整理、工作流說明與簡報圖片為主，避免綁定特定組織的內部資料。

## 1. 固定圖片風格提示詞

適合用在技術簡報、流程圖、架構圖、比較圖、roadmap、成果頁。

```text
請根據以下內容製作 16:9 資訊圖片，套用「白底極簡企業簡報風格」。

風格要求：
- 白色或極淺色背景
- 深藍主標題
- 企業技術簡報風
- 顧問式排版
- 圓角卡片
- 少量線性圖示
- 不要照片
- 不要複雜背景
- 不要過多文字
- 每張圖只保留核心訊息
- 使用繁體中文

版型請依內容自動選擇：
1. 左右對比
2. 三階段流程
3. 2x2 或 4x2 卡片
4. 大數字成果頁
5. 簡化架構圖

請優先整理與收斂內容，不要逐字放上去。
重點是讓讀者一眼理解，不是完整保留原文。

內容如下：
【貼上內容】
```

## 4. Codex App 使用場景提示詞

## 5. Codex 深層 Review 提示詞

適合介紹如何用 Codex Review 另一個 AI 或開發工具產生的程式碼、規格與文件。

```text
請幫我製作一組「Codex 深層 Review」簡報資訊圖片。

背景：
我會使用 Codex 對其他工具產出的程式碼、規格與文件進行深層 Review。

簡報目標：
- 說明為什麼需要第二意見 Review
- 說明 Codex 如何從質疑、找風險、補測試的角度檢查產出
- 說明 Review 後如何改善交付品質

請產生以下頁面：
1. 為什麼需要 Codex Review
2. Review 運作流程
3. Review 重點：規格一致性、邊界條件、命名與架構、測試性、風險
4. Review 輸出：問題清單、風險分級、修正建議、優先順序
5. 最佳使用流程：產出初稿 → Codex Review → 修正 → 再 Review

風格要求：
- 套用白底極簡企業簡報風格
- 16:9
- 白底、深藍標題、圓角卡片
- 少量圖示
- 不要太多文字
- 可直接貼到投影片
```

風格要求：

- 16:9
- 白底、深藍標題、圓角卡片
- 少量線性圖示
- 文字精簡
- 不要逐字貼原文

## 7. Codex × GPT Image 2 協作流程提示詞

適合做成總結頁或導入建議頁。

```text
請製作一張「Codex × GPT Image 2 協作流程」資訊圖片。

請表達以下流程：
1. 先用 Codex 讀取專案資料與需求
2. Codex 整理成結構化內容、流程、文件或程式
3. 若是程式碼、規格或重要文件，使用 Codex Review 做品質檢查
4. 修正後產出穩定內容
5. 再用 GPT Image 2 轉成簡報圖片、架構圖、流程圖或情境圖
6. 最後交付給讀者、主管、社群或專案團隊溝通

核心訊息：
內容由 Codex 整理，表達由 GPT Image 2 強化。

風格：
- 白底極簡企業簡報風格
- 16:9
- 流程式排版
- 少量圖示
- 文字要短
```

- 技術詞要保留，但要講得好懂

## 10. 圖片微調常用提示詞

### 移除多餘文字

```text
請保留這張圖的原風格與版型，但把文字再精簡 40%。
每張卡片只留標題與一句短說明。
不要新增內容。
```

### 拆成多張

```text
這張圖資訊太多，請拆成 3 張圖。
每張圖只講一個主題，風格保持一致。
```

### 改成更正式

```text
請把這張圖調整成正式簡報版。
移除可愛或裝飾性元素，保留白底、深藍標題、圓角卡片與少量線性圖示。
```

### 統一風格

```text
請將這張圖重畫成白底極簡企業簡報風格。
不要照抄原排版，請重新整理成更適合簡報的資訊圖。
```

### 改成決策者版本

```text
請把這張圖改成決策者版本。
減少技術細節，保留決策重點、效益、成本、風險控管與下一步。
```
