# Codex × GPT Image 2 圖片生成提示詞分享版

這份文件只保留和「圖片生成」直接相關的提示詞，作為 GPT Image 2 產生簡報圖片、技術資訊圖、流程圖、架構圖與視覺化成果的參考。

使用方式：直接複製需要的提示詞，替換中括號內容即可。範例以公開技術分享、專案案例整理、工作流說明與簡報圖片為主，避免綁定特定組織的內部資料。

- 白色或極淺色背景
- 深藍主標題
- 企業技術簡報風
- 顧問式排版
- 圓角卡片
- 少量線性圖示
- 不要照片
- 不要複雜背景
- 不要過多文字
- 每張圖只保留核心訊息
- 使用繁體中文

版型請依內容自動選擇：
1. 左右對比
2. 三階段流程
3. 2x2 或 4x2 卡片
4. 大數字成果頁
5. 簡化架構圖

請優先整理與收斂內容，不要逐字放上去。

## 5. Codex Review 圖片

適合把「深層 Review」流程畫成簡報資訊圖。

```text
請幫我製作一組「Codex 深層 Review」簡報資訊圖片。

背景：
我會使用 Codex 對其他工具產出的程式碼、規格與文件進行深層 Review。

簡報目標：
- 說明為什麼需要第二意見 Review
- 說明 Codex 如何從質疑、找風險、補測試的角度檢查產出
- 說明 Review 後如何改善交付品質

請產生以下頁面：
1. 為什麼需要 Codex Review
2. Review 運作流程
3. Review 重點：規格一致性、邊界條件、命名與架構、測試性、風險
4. Review 輸出：問題清單、風險分級、修正建議、優先順序
5. 最佳使用流程：產出初稿 → Codex Review → 修正 → 再 Review

風格要求：
- 套用白底極簡企業簡報風格
- 16:9
- 白底、深藍標題、圓角卡片
- 少量圖示
- 不要太多文字
- 可直接貼到投影片
```

- 16:9

- 少量圖示
- 文字要短

## 9. 圖片微調常用提示詞

### 移除多餘文字

```text
請保留這張圖的原風格與版型，但把文字再精簡 40%。
每張卡片只留標題與一句短說明。
不要新增內容。
```

### 拆成多張

```text
這張圖資訊太多，請拆成 3 張圖。
每張圖只講一個主題，風格保持一致。
```

### 改成更正式

```text
請把這張圖調整成正式簡報版。
移除可愛或裝飾性元素，保留白底、深藍標題、圓角卡片與少量線性圖示。
```

### 統一風格

```text
請將這張圖重畫成白底極簡企業簡報風格。
不要照抄原排版，請重新整理成更適合簡報的資訊圖。
```

### 改成決策者版本

```text
請把這張圖改成決策者版本。
減少技術細節，保留決策重點、效益、成本、風險控管與下一步。
```
